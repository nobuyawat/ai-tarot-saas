## 台本

皆さん、L13の全体テストでエラーが出た方もいるかもしれません。ここで大切なことをお伝えします。エラーが出ることは、開発では普通のことです。プロのエンジニアも毎日エラーと向き合っています。違いは何か？「エラーの解決方法を知っているかどうか」です。

そしてAI時代の開発では、エラーの解決方法はとてもシンプルです。エラーメッセージをそのままAIに貼り付ける。これだけです。

実際にやってみましょう。

### デモ1: 典型的なエラー

まず、わざとエラーを起こしてみます。ターミナルで、ai-tarot-saasフォルダの外で「npm run dev」と入力してみてください。すると「ENOENT: no such file or directory」というエラーが出ます。英語ですが、翻訳する必要はありません。このエラーメッセージをそのままコピーして、Claude Codeにこう貼り付けます。「このエラーが出ました。修正してください：」に続けて、エラーメッセージを貼り付けます。

するとAIが「ai-tarot-saasフォルダに移動してから実行してください」と教えてくれます。「cd ai-tarot-saas」で移動して、もう一度「npm run dev」。今度は正常に起動しますね。

### デモ2: ブラウザのエラー

次に、ブラウザで表示されるエラーを解決してみましょう。開発中に画面が真っ白になったり、赤い文字でエラーが表示されることがあります。こういう時も同じです。エラーメッセージをコピーして、Claude Codeに貼り付けるだけ。

画面が真っ白でエラーメッセージが見えない場合は、ブラウザの開発者ツールを開きます。F12キーを押すか、右クリックして「検証」を選んでください。Consoleタブに赤い文字でエラーが表示されています。これをコピーしてAIに渡せば、修正方法を教えてくれます。

### デモ3: 見た目の問題

エラーではないけれど、見た目がおかしい場合もAIに頼めます。例えば「テキストがはみ出している」「ボタンの位置がずれている」「スマホで表示が崩れる」。こういう時は、問題を日本語で具体的に伝えます。

ポイントは「具体的に伝えること」です。「おかしいです」ではなく、「カードをクリックしても裏返らない」。「動きません」ではなく、「/loginに遷移せずに画面が真っ白になる」。具体的であればあるほど、AIは正確に修正してくれます。

### エラー解決の3ステップ

まとめます。エラーが出たときの手順は、たった3つです。

1つ目。エラーメッセージをコピーする。ターミナルでもブラウザでも、赤い文字をそのままコピーしてください。

2つ目。Claude Codeに「このエラーが出ました。修正してください：」と書いて、エラーメッセージを貼り付ける。

3つ目。AIが修正してくれたら、もう一度動作を確認する。

これだけです。エラーを自分で読み解く必要はありません。英語が分からなくても大丈夫。AIが全部やってくれます。

AI時代の開発スキルとは、「エラーを自分で直す力」ではなく、「エラーをAIに正確に伝える力」なんです。この講座を通じて、皆さんはすでにそのスキルを身につけています。日本語でAIに指示を出して、アプリを作ってきましたよね。エラー解決も同じことです。

もしこの先、講座以外で自分のアプリを作るときにエラーが出ても、もう怖くありません。AIがあなたのパートナーです。では、次のレクチャーでいよいよアプリを世界に公開しましょう！
